Git Repo Auth — Promotional Images: Paired Prompts
====================================================

Each card visualizes real output from the connector, captured during
pre-submission validation against production (v0.3.0). The prompt that
produces each output in Claude:

card-1.png — "Call github_token with no arguments and show me the response."
  (Bare mint: read-only default, ≤1h expiry, quota block in every reply.)

card-2.png — "Use that token to push my branch."
  (GitHub refuses the read-only token with a 403 naming git-repo-auth[bot] —
  the secure default and the audit identity, demonstrated by a denial.)

card-3.png — "Mint the same scope again — what does that cost me?"
  (Same-scope re-request within a live token's lifetime: cached, free,
  reported truthfully in the quota block.)

card-4.png — "What happens if I uninstall the GitHub App?"
  (The kill switch: all minting stops instantly, enforced by GitHub.
  The card enumerates every wall GitHub enforces.)

All cards: 1600×1000 PNG, brand palette (pine #08322A, viridian #0E5A4A,
paper #FAFAF6, amber #D9A441), Archivo + IBM Plex Mono.
